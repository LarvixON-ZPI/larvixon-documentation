## Architektura i technologie

Aplikacja kliencka została zrealizowana we frameworku **Flutter** (wsparcie dla Web, Android, Windows). Zastosowano architekturę warstwową inspirowaną **Clean Architecture**:

1.  **Data Layer**: Repozytoria, DTO, Dio (HTTP).
2.  **Domain Layer**: Encje biznesowe.
3.  **Presentation Layer**: UI oraz BLoC.

![Komponenty frontend](../../../../../assets/images-frontend/frontend-componentsFrontendComponents.png)

## Wzorce projektowe

- **BLoC (Business Logic Component)**: Separacja logiki od UI, zarządzanie stanem.
- **Repository**: Abstrakcja źródeł danych.
- **Dependency Injection**: Zarządzanie zależnościami.
- **Factory**: Tworzenie obiektów błędów (`ApiFailure`).
