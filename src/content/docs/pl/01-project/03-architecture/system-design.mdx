## Repozytoria kodu

System został podzielony na niezależne komponenty hostowane w organizacji GitHub [LarvixON-ZPI](https://github.com/LarvixON-ZPI). Główne repozytoria to:

- Backend (Django)
- Frontend (Flutter)
- Model ML (PyTorch)
- Serwis Pacjentów (FastAPI)
- Symulacja (Unity)

## Architektura ogólna

Architektura backendu została zaprojektowana zgodnie z wzorcem modułowego monolitu, oddzielając logikę biznesową od procesów obliczeniowych.

### Diagram kontenerów (C2)

System składa się z następujących elementów infrastruktury:

1.  **Django App:** API i logika biznesowa.
2.  **Celery Worker:** Asynchroniczne przetwarzanie wideo.
3.  **Redis:** Broker wiadomości i cache.
4.  **PostgreSQL:** Relacyjna baza danych.
5.  **Nginx:** Reverse proxy.

![Diagram kontenerów](../../../../../assets/images-backend/larvixon-Containers.png)

## Infrastruktura chmurowa (Azure)

System wykorzystuje usługi zarządzane (Managed Services) w celu zapewnienia skalowalności:

- **Azure Database for PostgreSQL**: Przechowywanie metadanych.
- **Azure Cache for Redis**: Obsługa kolejki zadań.
- **Azure Blob Storage**: Magazyn plików wideo (integracja przez `django-storages`).
